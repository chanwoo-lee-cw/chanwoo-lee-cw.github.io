Project 01 · 바잉스퀘어 · 2025.03 – 2025.06

Python Flask 레거시 프로젝트의 Kotlin 마이그레이션

👥 총 11명 (기획 3, 백엔드 5, 프론트 3) 🔧 Kotlin · Spring Boot · Kafka · MySQL · Flask
개요

기존 레거시 프로젝트는 특정 에러 처리 로직을 사용하기 위해 상속·팩토리 메서드·메서드 리플렉션이 지나치게 많이 사용된 구조였습니다. 작업 흐름이 복잡해지고 사이드 이펙트가 자주 발생하여 유지보수와 테스트가 매우 어려웠습니다.

이를 개선하기 위해 정적 타입 기반 언어인 Kotlin/Java 전환을 제안했고, Layered 아키텍처 구조와 null-safe한 설계로 Spring Boot 기반 시스템으로 마이그레이션을 진행했습니다.

목표
  • Python Flask 기반 레거시를 Kotlin Spring Boot 구조로 재설계
  • 사이드 이펙트 파악이 어려운 복잡한 구조를 명시적인 호출 구조로 변경
  • 불명확한 필드명을 명확하게 정비하는 방향으로 마이그레이션
  • 기존 Flask 프로젝트에 함께 있던 Email 기능을 별개의 서버로 분리

셀러 오더 관리를 위한 셀러 어드민 시스템 구축

👥 총 8명 (기획 2, 백엔드 3, 프론트 3) 🔧 Kotlin · Kafka · Redis · SSE
개요

기존 카탈로그 시스템에서는 셀러가 주문 제출 시 입력한 정보만 확인할 수 있었습니다. 이후 오더 컨펌·오더 취소 상태를 확인할 수 없는 상황이었습니다.

셀러들이 자사 브랜드의 오더 현황을 실시간으로 확인하거나 바이어별로 세부적인 상태를 파악하기 어려운 문제를 해결하기 위한 어드민 시스템 구축 프로젝트입니다. 브랜드 가치 유지를 위해 신뢰할 수 있는 바이어 확인 및 주문 제어·관리 기능을 제공하는 것이 목표였습니다.

목표
  • 내부 운영자는 카탈로그 어드민을 통해 오더 상태(오더 컨펌, 결제/배송, 오더 완료, 오더 취소)를 확인하고 처리
  • 단순 상태 변경만 지원하고, 기존 레거시에서 처리하던 비즈니스 과정은 기존 시스템을 통해 진행되도록 설계
  • 카탈로그 어드민 내의 새로운 주문 관리 페이지 생성
  • 카탈로그 DB 내에서 직접 재고 검증 가능하고, 장기적으로 분리할 수 있도록 설계
